KEY ARCHITECTURE HIGHLIGHTS
───────────────────────────

U-NET SKIP CONNECTIONS:
   "The skip connections in U-Net are crucial for image denoising. They allow
    low-level spatial information from the encoder to bypass the bottleneck,
    preventing information loss during downsampling."

ENCODER PATH (Downsampling):
   Input (1×64×64) → 64 → 128 → 256 → 512 channels
   Each followed by MaxPool for spatial reduction

BOTTLENECK (Feature Extraction):
   512 → 1024 → 512 channels
   Maximum feature compression before decoder

DECODER PATH (Upsampling + Skip):
   512 → 256 → 128 → 64 channels
   Each layer concatenates skip connections from encoder

OUTPUT LAYER:
   64 → 3 channels (RGB output)
   No activation (raw predictions for denoising)

COMMON QUESTIONS ANSWERED
──────────────────────────

Q: Why U-Net instead of other architectures?
A: U-Net has skip connections that preserve spatial information crucial
   for image denoising. Industry standard for this task.

Q: Can I use this with other datasets?
A: Yes. The model is flexible. Just adjust image_size in config.yaml
   and ensure data is normalized to [0, 1].

Q: What if I don't have a GPU?
A: It will work on CPU but be slower (~8 hours vs 1 hour training).
   Set device: "cpu" in config.yaml.

Q: How do I know if my noise is realistic?
A: Use visualizations. If denoised output looks like original, noise
   is realistic. Compare with before/after quality.

Q: Can I use different loss functions?
A: Yes. config.yaml has options: 'mse', 'l1', or 'combined'.
   CombinedLoss is recommended for faster convergence.

Q: What if validation loss increases?
A: Sign of overfitting. Try: reduce learning_rate, increase
   regularization (weight_decay), use early stopping.
